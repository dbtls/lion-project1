# 프로젝트 설계

## 1. 서비스 설명

사용자가 할 일을 등록하고 완료 여부를 관리할 수 있는 간단한 Todo 관리 서비스.

## 2. 전체 흐름

사용자는 프론트엔드 화면에서 할 일을 입력하고 등록한다.

프론트엔드는 입력값을 검증한 뒤 백엔드 API로 요청을 전송한다.

백엔드는 요청을 처리하여 MySQL 데이터베이스에 데이터를 저장하거나 수정하고,

처리 결과를 다시 프론트엔드로 반환한다.

프론트엔드는 응답 데이터를 기반으로 화면을 갱신하여 사용자에게 최신 상태를 보여준다.

## 3. 프론트엔드 역할

- Todo 목록 조회 요청 전송
- Todo 등록, 완료 처리, 삭제 요청 전송
- 사용자 입력값 검증 및 UI 상태 관리
- 백엔드 응답 데이터를 기반으로 화면 렌더링

## 4. 백엔드 역할

- Todo 관련 REST API 제공
- 데이터 유효성 검증 및 비즈니스 로직 처리
- MySQL 데이터베이스와의 데이터 저장 및 조회
- 프론트엔드 요청에 대한 JSON 형태의 응답 반환

## 5. 배포 구조 요약

프론트엔드, 백엔드에 각각의 Dockerfile을 생성하고, docker-compose.yml을 통해 각각의 컨테이너를 실행시킨다.

AWS EC2 인스턴스에 Docker와 Docker Compose를 설치하고, 동일한 Docker Compose 구성을 사용하여 서비스를 배포한다.
